# Contributing to DiscordRPG

Thank you for your interest in contributing to DiscordRPG! Here are some guidelines to help you get started.

## Types of Changes

### Wanted Changes
- Fixes to bugs or issues within the code.
- Enhancements or new features that improve the functionality of the bot.
- Improvements to documentation or readability of the code.
- Performance optimizations.

### Unwanted Changes
- Whitespace or formatting changes that do not improve readability.
- Subjective changes to coding style without a significant reason.
- Major restructuring of the code without prior discussion.

## How to Contribute

### Reporting Issues
1. Search the existing issues to ensure your issue has not been reported.
2. Open a new issue and provide a clear, concise description of the problem.
3. Include steps to reproduce the issue, if applicable.
4. Provide any relevant screenshots or logs.

### Submitting Pull Requests
1. Fork the repository and create a new branch from `master`.
2. Make your changes in the new branch.
3. Ensure your code follows the project's coding standards.
4. Test your changes thoroughly.
5. Submit a pull request with a clear description of your changes and the problem they solve.
6. Link to any relevant issues in your pull request.

## Code of Conduct
This project adheres to a [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code. Please report unacceptable behavior to [karpadf@gmail.com].
